You will receive:
1) The original input text
2) The Stage-1 strict extraction JSON

GOAL
Propose conservative PWML-constructible additions that improve connectivity and completeness while staying grounded in the text.

CRITICAL RULES
- You may infer missing links, but be conservative.
- Every inferred object must include:
  (a) at least one supporting evidence quote (<=25 words, verbatim from the original text),
  (b) top-level fields: "confidence" (0.0–1.0), "provenance": "inferred", "source_refs": [array of notes/evidence strings]
- Do not use the internet. Do not introduce unsupported facts.
- Never overwrite Stage-1 objects. Only add objects/links.
- If unsure, omit. Prefer missing data over hallucination.
- Output only a single JSON object. No prose, no markdown.
- Processes must be under additions.processes, not additions.entities.
- Do not emit empty placeholder objects or empty arrays.
- Do not add composite entity names containing '+' or ' and '.
- For expressions like "A + B -> C", place A and B as separate inputs, never as a single entity token.
- Transport cargo must be a single entity token. If text implies multiple cargos, add separate transport rows.

ALLOWED INFERENCES
A) Enzyme to reaction linkage
- If text implies catalysis, attach an enzyme to an existing reaction.
- If no reaction exists and conversion is explicit, you may add a new reaction.

B) Reactions (only when conversion is explicit)
Allowed patterns:
- "X is synthesized from A, B"
- "A and B form C"
- "enzyme converts A to B"
- "A is converted into B"
Constraints:
- A reaction must have >=1 input and >=1 output.
- Inputs/outputs should use existing Stage-1 entity names when possible.
- Do not create reactions from vague statements.

C) Alias/synonym bridging for connectivity
- If explicit equivalence exists (example: "Glutathione (GSH)"), add an interaction:
  (entity_1, relationship=SAME_AS, entity_2)

D) Minimal protein_complex wrapper
- If needed for enzyme attachment, you may add a single-protein complex.

E) Biological state and location linking
- You may add biological_states and element_locations when explicitly supported.
- Prefer linking existing entities to states over inventing new entities.
- If an inferred product/input phrase is composite, split it into multiple entities and keep the process stoichiometry explicit.

F) Use graph QA feedback conservatively
- Treat QA feedback as repair hints only.
- Prioritize reconnecting disconnected entities using supported reactions, links, and state/location assignments.

NOT ALLOWED
- No species/cell type/tissue/subcellular location unless explicitly stated.
- No taxonomy/ontology IDs unless explicitly stated.
- No pathway steps not supported by text.
- No diagram/layout coordinate inventions.
